# Agent Instructions for sampaloc-iv-mis

## Overview
This project uses a modern data-fetching architecture combining:
- **Axios** for HTTP requests with consistent configuration
- **TanStack Query (React Query)** for client-side caching, synchronization, and state management
- **Next.js Server Actions** for mutations (POST, PATCH, DELETE)
- **TanStack Query hooks** for managing loading states, errors, and data synchronization

## 1. API Calls with Axios

### Configuration
Always use the pre-configured axios instance from `lib/axios.ts`:
```typescript
import { axiosInstance } from '@/lib/axios';
```

### GET Requests (TanStack Query)
Use `useQuery` for all GET requests:
```typescript
import { useQuery } from '@tanstack/react-query';
import { axiosInstance } from '@/lib/axios';

const { data, isLoading, error } = useQuery({
  queryKey: ['residents', id],
  queryFn: async () => {
    const { data } = await axiosInstance.get(`/api/residents/${id}`);
    return data;
  },
  staleTime: 1000 * 60 * 5, // 5 minutes
});
```

## 2. Server Actions for Mutations

### POST - Create
Use server actions for POST requests:
```typescript
// app/actions/residents.ts
'use server';

import { axiosInstance } from '@/lib/axios';

export async function createResident(formData: any) {
  try {
    const { data } = await axiosInstance.post('/api/residents', formData);
    return { success: true, data };
  } catch (error: any) {
    return { success: false, error: error.response?.data?.message || 'Failed to create resident' };
  }
}
```

Client usage with `useMutation`:
```typescript
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { createResident } from '@/app/actions/residents';

const queryClient = useQueryClient();
const mutation = useMutation({
  mutationFn: createResident,
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ['residents'] });
  },
});
```

### PATCH/PUT - Update
Use server actions for update operations:
```typescript
// app/actions/residents.ts
'use server';

export async function updateResident(id: string, formData: any) {
  try {
    const { data } = await axiosInstance.patch(`/api/residents/${id}`, formData);
    return { success: true, data };
  } catch (error: any) {
    return { success: false, error: error.response?.data?.message || 'Failed to update resident' };
  }
}
```

Client usage:
```typescript
const updateMutation = useMutation({
  mutationFn: ({ id, data }: { id: string; data: any }) => updateResident(id, data),
  onSuccess: (_, variables) => {
    queryClient.invalidateQueries({ queryKey: ['residents', variables.id] });
  },
});
```

## 3. TanStack Query Patterns

### Handling Loading, Error, and Caching

```typescript
const { data, isLoading, error, isFetching } = useQuery({
  queryKey: ['residents', filter],
  queryFn: async () => {
    const { data } = await axiosInstance.get('/api/residents', { params: { filter } });
    return data;
  },
  staleTime: 1000 * 60 * 5,       // Data fresh for 5 min
  gcTime: 1000 * 60 * 10,          // Keep in cache for 10 min
  retry: 2,                         // Retry failed requests twice
  refetchOnWindowFocus: true,       // Refetch when window regains focus
});
```

### Mutation with Error Handling
```typescript
const mutation = useMutation({
  mutationFn: async (data: ResidentData) => {
    const { data: response } = await axiosInstance.post('/api/residents', data);
    return response;
  },
  onMutate: async (newData) => {
    // Optimistic update
    await queryClient.cancelQueries({ queryKey: ['residents'] });
    const previousResidents = queryClient.getQueryData(['residents']);
    queryClient.setQueryData(['residents'], (old: any) => [...old, newData]);
    return { previousResidents };
  },
  onError: (error: any, newData, context: any) => {
    // Rollback on error
    queryClient.setQueryData(['residents'], context.previousResidents);
  },
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ['residents'] });
  },
});
```

## 4. Component Implementation Example

```typescript
'use client';

import { useMutation, useQuery, useQueryClient } from '@tanstack/react-query';
import { createResident, updateResident, deleteResident } from '@/app/actions/residents';
import { axiosInstance } from '@/lib/axios';

export function ResidentForm({ id }: { id?: string }) {
  const queryClient = useQueryClient();

  // Fetch if editing
  const { data: resident, isLoading: isLoadingResident } = useQuery({
    queryKey: ['residents', id],
    queryFn: async () => {
      const { data } = await axiosInstance.get(`/api/residents/${id}`);
      return data;
    },
    enabled: !!id, // Only fetch if id exists
  });

  // Mutation for create/update
  const mutation = useMutation({
    mutationFn: (formData: any) => id ? updateResident(id, formData) : createResident(formData),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['residents'] });
    },
  });

  return (
    <form onSubmit={(e) => {
      e.preventDefault();
      const formData = new FormData(e.currentTarget);
      mutation.mutate(Object.fromEntries(formData));
    }}>
      {isLoadingResident && <p>Loading...</p>}
      {mutation.isPending && <p>Submitting...</p>}
      {mutation.error && <p>Error: {mutation.error.message}</p>}
      {/* Form fields */}
      <button type="submit" disabled={mutation.isPending}>
        Submit
      </button>
    </form>
  );
}

export function ResidentList() {
  const queryClient = useQueryClient();
  const { data: residents, isLoading, error } = useQuery({
    queryKey: ['residents'],
    queryFn: async () => {
      const { data } = await axiosInstance.get('/api/residents');
      return data;
    },
  });

  const deleteMutation = useMutation({
    mutationFn: deleteResident,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['residents'] });
    },
  });

  if (isLoading) return <div>Loading residents...</div>;
  if (error) return <div>Error loading residents</div>;

  return (
    <ul>
      {residents?.map((resident: any) => (
        <li key={resident.id}>
          {resident.name}
          <button
            onClick={() => deleteMutation.mutate(resident.id)}
            disabled={deleteMutation.isPending}
          >
            Delete
          </button>
        </li>
      ))}
    </ul>
  );
}
```

## 5. Query Key Factory Pattern

Organize query keys consistently:
```typescript
// lib/query-keys.ts
export const residentsQueryKeys = {
  all: ['residents'] as const,
  lists: () => [...residentsQueryKeys.all, 'list'] as const,
  list: (filters?: any) => [...residentsQueryKeys.lists(), filters] as const,
  details: () => [...residentsQueryKeys.all, 'detail'] as const,
  detail: (id: string) => [...residentsQueryKeys.details(), id] as const,
};
```

Usage:
```typescript
const { data } = useQuery({
  queryKey: residentsQueryKeys.detail(id),
  queryFn: () => fetchResident(id),
});

queryClient.invalidateQueries({ queryKey: residentsQueryKeys.all });
```

## 6. Error Handling Best Practices

```typescript
// lib/error-handler.ts
export function handleApiError(error: any): string {
  if (error.response?.data?.message) {
    return error.response.data.message;
  }
  if (error.message) {
    return error.message;
  }
  return 'An unexpected error occurred';
}
```

## 8. Critical Rules

1. **All GET requests** must use `useQuery`
2. **All POST/PATCH/DELETE** must use server actions + `useMutation`
3. **Always invalidate queries** on successful mutations
4. **Always handle loading and error states**
5. **Use query keys factory** for consistency
6. **Set appropriate staleTime and gcTime**
7. **Implement optimistic updates** for better UX
8. **Use `enabled` option** to conditionally fetch data
9. **Never call server actions from server actions** (leads to complexity)
10. **Always import from `lib/axios`** for consistent configuration
